SECTION 8: CLIENT PROJECT REQUEST SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Clients can submit project requests directly from their user dashboard.

How to Submit a Project:
1. Log in → User Dashboard → My Projects
2. Click "New Project" / "+ Request Project"
3. Fill in project details:
   - Project Name
   - Project Type (Web Development / Mobile App / E-Commerce / Design / Consulting / Other)
   - Budget Range
   - Project Details / Description
   - Attachments (optional — PDF, DOC, images etc.)
4. Submit — admin receives instant notification

Project Status Flow:
pending → in_review → approved OR rejected → completed

Status Explanations:
- pending: Just submitted, awaiting admin review
- in_review: Admin is currently evaluating the project
- approved: Project accepted, work has started or will start soon
- rejected: Project not accepted (reason provided in admin notes)
- completed: Project successfully delivered

Progress Tracking:
- Percentage progress bar (0–100%) updated by admin or team
- Deadline date visible on each project
- Total cost and amount paid tracked

Payment Status (automatic):
- unpaid: No payment received yet
- partial: Some payment received
- paid: Fully paid

File Attachments:
- Clients can upload files (PDF, DOC, DOCX, images) with their request
- Files stored securely on Cloudinary
- Admin and team can view/download attachments

Archive: Completed or rejected projects can be archived.
Delete: Only pending or rejected projects can be deleted by the user.

Team Assignment:
- Admin assigns team members to each project
- Team members can see assigned projects in their /team/projects page
- Assigned team receives a real-time notification

─── 10.4 NOVA AI CAPABILITIES ────────────────────────────────────────────────
Intelligence Features:
- Greeting detection (instant responses, no API cost)
- Off-topic filter (politely declines non-business questions)
- FAQ keyword matching (answers from knowledge base, no API cost)
- Response caching (repeated questions answered instantly)
- Smart model routing (simple queries → fast Haiku model, complex → Sonnet)
- Rate limiting (15 requests per minute per IP — fair use)

Tone Options (Admin Configurable):
Professional | Friendly | Formal | Casual | Expert | Empathetic

Models Used:
- Simple queries: claude-haiku-4-5-20251001 (fast, cost-effective)
- Complex queries: claude-sonnet-4-6 (more capable, nuanced)

─── 10.5 KNOWLEDGE BASE ──────────────────────────────────────────────────────
Admin can add knowledge to Nova via /admin/chatbot-manager → Knowledge Base tab

Knowledge Types:
- text: Plain text or markdown content
- faq: Question & answer pairs
- file: Uploaded documents (PDF, TXT, MD, DOC, CSV, JSON)
- url: Website pages (server crawls and extracts text automatically)
- auto: Auto-synced from database (Services, Portfolio, Jobs, Team, etc.)

RAG (Semantic Search) System:
- Entries are embedded using OpenAI text-embedding-3-small (1536 dimensions)
- Stored in Supabase pgvector for semantic similarity search
- 3-layer retrieval: Semantic Search → Full-text Search → Recency Fallback
- Role-based access: public / user / team / admin knowledge tiers

Auto-Sync from Database:
Admin can sync the knowledge base automatically from:
- Services
- Portfolio Projects
- Team Members
- Active Job Listings
- Client Testimonials
- CMS Content (About, Process, Why-Us)
- Company Info

SECTION 22: ANNOUNCEMENT BAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A scrolling announcement ticker at the top of every public page.
Admin can create/edit/delete announcements from /admin/announcements.

Features:
- Multiple announcements in an infinite scrolling loop
- Each announcement can have: text, emoji, link, link label, background color, text color
- Speed is configurable (5–120 seconds per full scroll)
- Admin can enable/disable individual announcements
- Active announcements are visible to all visitors
- Inactive announcements are hidden from public but stay in the database
